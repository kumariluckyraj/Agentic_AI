2. Anatomy of an Agent Prompt

An agent prompt is usually much larger than a normal prompt.

A. System Prompt
Definition

High-level instructions controlling overall agent behavior.

It defines:

personality
rules
objectives
boundaries
workflow

B. Tool Prompt
Definition

Instructions specific to a tool.

Example

Suppose an agent has a SQL tool.

Tool prompt:

Generate safe SQL queries only.

Never:
- DROP tables
- DELETE without WHERE clause

Return:
- query
- explanation
Another Example

Browser tool prompt:

When browsing:
- summarize key information
- avoid ads
- extract only relevant data

5. Zero-Shot vs Few-Shot Prompting

One of the MOST important prompting concepts.

A. Zero-Shot Prompting
Definition

Giving a task without examples.

Example
Classify sentiment:
"I love this product."

B. Few-Shot Prompting
Definition

Providing examples before the actual task.

Example
Input: "Amazing service"
Output: Positive

Input: "Terrible experience"
Output: Negative

Input: "The food was cold"
Output:

Agents often use few-shot examples to:

improve tool usage
standardize reasoning
reduce hallucinations


Normal Prompt

Example Schema
{
  "type": "object",
  "properties": {
    "city": {
      "type": "string",
      "description": "City name"
    },
    "unit": {
      "type": "string",
      "enum": ["celsius", "fahrenheit"]
    }
  },
  "required": ["city"]
}
Meaning

This schema says:

input must be object
city must be string
unit must be one of allowed values

7. Why Should Tools Be Constrained?

VERY important interview question.

Answer

Tools must be constrained because LLMs are probabilistic systems.

They:

hallucinate
generate incorrect inputs
misunderstand instructions
optimize for plausible outputs


Most Important Takeaway

Agent intelligence is NOT just:

reasoning

It is:

reasoning
+ safe tool usage
+ validation
+ controlled execution



A. Breadth-First Search (BFS)
Definition

Explore all nearby possibilities before going deeper.

Visualization
Level 1:
A B C

Level 2:
A1 A2 B1 B2 C1


B. Depth-First Search (DFS)
Definition

Explore one branch deeply before trying others.

Visualization
A → A1 → A1a → A1b

before exploring B.

DFS Behavior
Go deep quickly
